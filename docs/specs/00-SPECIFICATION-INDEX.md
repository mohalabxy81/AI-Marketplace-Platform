## How To Use This Specification Set

### For Database Engineers
1. Start with **Spec 02** (Master Database Architecture) — all schema definitions
2. Read **Spec 03** (Multi-Tenant) — RLS patterns and isolation rules
3. Read **Spec 04** (Marketplace) — specific marketplace tables
4. Reference **Spec 15** (Security) — RLS policy patterns and DB role separation

### For Backend Engineers
1. Read **Spec 01** (Domain Decomposition) — understand service boundaries
2. Read **Spec 08** (Event Contracts) — all Kafka topics and payload schemas
3. Read **Spec 09** (API Contracts) — all REST/gRPC interface definitions
4. Read the domain-specific spec for your team's domain (05, 06, 11, 12, etc.)

### For Intelligence / ML Engineers
1. Read **Spec 05** (Discovery Engine) — 4-stage ranking pipeline
2. Read **Spec 06** (AI Systems) — model registry, gateway, token guard
3. Read **Spec 07** (Vector & Embedding) — embedding lifecycle, HNSW config

### For Frontend Engineers
1. Read **Spec 16** (Frontend Systems) — all 4 client apps, rendering strategy, design system
2. Read **Spec 09** (API Contracts) — API shapes the frontend consumes
3. Read **Spec 10** (Realtime) — WebSocket channel topology

### For SRE / Operations Engineers
1. Read **Spec 14** (Observability) — OTel standards, SLOs, alerting
2. Read **Spec 15** (Security) — WAF, TLS, secrets management
3. Read **Spec 20** (Final Blueprint) — scaling roadmap, operational checklist

### For Engineering Leadership
1. Read **Spec 17** (Engineering Execution Plan) — sprints, teams, dependencies
2. Read **Spec 18** (MVP Definition) — scope, launch criteria, version roadmap
3. Read **Spec 19** (Build Readiness) — risk register, Go/No-Go decision
4. Read **Spec 20** (Final Blueprint) — system map, laws, status
