# Phase AV-1: Agent Economy & Monetization Model

> **Document Status**: Draft
> **Phase**: AV-1 (Agent Economy)
> **Owner**: Strategy & Engineering

## 1. Executive Summary

Phase AV introduces the **Agent Economy**, transforming our platform from a first-party closed system into an open, multi-sided ecosystem where third-party developers can publish their own autonomous agents, and tenants can subscribe to or purchase these agents to optimize their operations.

## 2. Monetization Model

### 2.1 The Revenue Split (70/30)
- Developers who publish an agent receive **70%** of the gross revenue generated by their agent.
- The Platform retains **30%** to cover inference costs (LLMs), sandbox execution infrastructure, and marketplace discovery placement.

### 2.2 Billing Mechanics
- **Subscription Model**: Agents can charge a flat monthly fee (e.g., $49/mo).
- **Consumption Model**: Agents can charge per successful action (e.g., $0.05 per support ticket resolved).
- All agent billing is aggregated into the tenant's existing platform invoice via Stripe Connect.

## 3. The Token Ledger Update
The existing `billing_ledger` will be updated to support third-party payee routing:
```sql
ALTER TABLE billing_ledger ADD COLUMN payee_account_id uuid NULL;
ALTER TABLE billing_ledger ADD COLUMN platform_fee_amount decimal(10,2) NULL;
```

## 4. Agent Quality & Risk Scoring
Agents are continually monitored by the `Risk Engine` (Phase AU). An agent's Trust Score impacts its ranking in the Agent App Store. Agents that generate a high number of rollbacks or trigger security flags will have their revenue withheld and face automatic delisting.
